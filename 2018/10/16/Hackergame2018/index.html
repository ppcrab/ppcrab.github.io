在程序初始化阶段进行操作~

使用gcc 的 attribute((constructor)) 一系列操作可以做到在程序初始化阶段进行一些操作。
比如这个C语言作业题目就是在程序初始化阶段就定义了一些代码。

触发 SIGFPE 的情况

不仅除以 0 可能会触发 SIGFPE，最小的 INT 除以 -1 也可能会触发。这就很难受了，我知道这个题目的第一步就是去将异常情况触发，然后运行_err函数，但是。。。我不知道怎么弄。现在知道了，居然除了除以0，还能 -2147483648/-1。吐血

linux命令。

进入vim命令行中之后，可以使用esc退出，然后输入‘：’，就可以输入想要运行的命令了，输入完毕后直接回车。cat - 中的 - 表示标准输入，cat 会试图从你的标准输入中读取，并不能看到 - 文件的内容。绕过的办法有多种，最简单的就是 cat ./-。不得不说linux命令还是很值的学习的。
除此之外还有好几种方式可以达到执行linux命令的结果，比如进入linux下的gdb模式。就可以输入命令了，相比之下，还比进入vim更快呢。

逆向

逆向的赛题是有一定的思路的，一般我们先会运行一下这个程序，看看主函数一开始干了什么，然后去找主函数，看看有没有什么发现。然后再去找字符串，从字符串入手，如果到现在还是没有十分明确的思路，那么只能从系统函数入手，一般加密的程序会涉及到字符串的一些操作，比如字符串倒转啊strrev(),字符串比较啊，字符串复制啊。
